## 프로젝트 개요
<b>누비다(NUBIDA)</b>는 SDGs 8번 "좋은 일자리와 경제 성장"을 주제로<br>
사용자 맞춤 여행 일정 생성과 체험형 단기 일자리 매칭을 결합한 관광 SNS 플랫폼입니다.<br>
인기 관광지 쏠림 현상을 완화하고, 숨겨진 관광지를 발굴하여 지역 경제 활성화와 균형 발전을 목표로 합니다.

레포지토리 위치 : <a href="https://github.com/gyuwon09/NUBIDA">gyuwon09/NUBIDA</a>

## 기술 스택
- **Language**: Python  
- **Frontend**: ReactJS, HTML5, CSS3, JavaScript  
- **Backend**: FastAPI
- **AI / Data**: LLM 기반 NLP, 퍼셉트론 분류 알고리즘, 관광 빅데이터 분석  
- **Platform**: Web

## 팀원 구성
- **총괄 / 백엔드**: 황규원
- **디자인**: 길지연
- **프론트엔드**: 길지연, 장찬민
- **데이터베이스 확보**: 이연우, 박현승

## 주요 기능
<b>누비다(NUBIDA)</b>의 기능은 **SNS · 여행 일정 생성 · 일자리 탐색** 세 가지로 구분됩니다.

### 나의 여행 SNS 플랫폼
<b>누비다(NUBIDA)</b>는 사용자가 여행 일정을 생성·공유할 수 있는 SNS 기반 관광 플랫폼입니다.<br>
사용자는 자신의 여행 일정과 관광지 경험을 사진과 함께 업로드하여 다른 사용자들에게 정보를 제공하고, 여행 영감을 공유할 수 있습니다.<br>
또한 관광지별 점수 산정 시스템을 도입하여, 상대적으로 덜 알려진 지역의 숨은 명소가 사용자에게 효과적으로 노출되도록 설계했습니다.

특히 사용자 게시글은 단순 후기 데이터에 그치지 않고 **관광지 평가 데이터**로 활용됩니다.<br>
사용자가 작성한 콘텐츠를 규칙 기반 평가 기준에 따라 정제한 뒤, LLM 기반 자연어 분석을 통해 관광지의 특성을 평가합니다.<br>
이 결과를 정량 점수로 변환하여 관광지 추천 알고리즘의 우선순위 산정에 반영함으로써, 데이터 기반의 신뢰도 높은 추천이 가능하도록 구현했습니다.

**관광지식정보시스템 국내관광통계** 등의 공공 관광 데이터를 함께 활용했습니다.

### AI 여행 일정 생성기
사용자의 여행 선호도를 설문 형태로 수집·분석하여 개인의 여행 유형을 도출합니다.<br>
분석 결과를 바탕으로 관광지 데이터베이스에서 최적의 여행지를 선별하고, 맞춤형 여행 일정을 자동 생성하도록 구현했습니다.<br>
또한 명소 발굴 알고리즘을 함께 적용하여, 가치가 높다고 판단된 비인기 관광지를 우선 추천함으로써 관광 수요 분산과 지역 균형 발전을 유도합니다.

<a href="https://youtu.be/3ASpGfAQrzs">일정 생성 프로토타입 시연 (Youtube)</a>

생성된 여행 일정은 설정 화면에서 언제든지 다시 열람할 수 있도록 했습니다.<br>
또한 사용자의 일정 생성 기록과 업로드한 여행 데이터를 기반으로 **퍼셉트론 기반 분류 함수**를 적용하여 사용자를 5가지 여행 유형으로 분류합니다.<br>
이 분류 결과는 이후 개인 맞춤형 여행지 추천과 체험형 일자리 매칭의 정확도를 높이는 데 활용됩니다.

### 일자리 탐색
관광지 및 지역 행사와 연계된 **단기 일자리·체험형 일자리 매칭 기능**을 제공합니다.<br>
이를 통해 지역 기반 일자리 접근성을 높이고, 지역 경제 활성화에 기여하도록 설계했습니다.

일자리는 이전 퍼셉트론 기반 분류 함수를 통해 나눠진 사용자 유형에 따라 추천되도록 설계했습니다.<br>

일자리 공고는 별도의 **일자리 탐색 대시보드 웹사이트**를 통해 등록·관리할 수 있습니다.<br>
관리자는 공고 등록, 수정, 삭제가 가능하며 지원자 정보 조회 기능을 통해 효율적인 인력 관리가 가능하도록 구현했습니다.

## 기능 구현

### 서비스 기능
본 프로젝트의 백엔드는 **Python 기반 FastAPI**로 구축하였으며, 
실제 서비스 흐름과 유사한 서비스 기능을 갖추도록 구현했습니다.

- **FastAPI**를 활용한 RESTful API 서버 설계 및 구현  
- **googlemaps** 라이브러리를 이용한 관광지 위치 및 정보 처리  
- **pandas** 기반 데이터 전처리 및 분석 파이프라인 구축  
- **OpenAI API** 연동을 통한 사용자 게시글 자연어 분석 및 여행 추천 로직 구현  

또한 사용자 인증 보안을 위해 다음 기능을 적용했습니다.

- **JWT(Json Web Token)** 기반 인증/인가 시스템 구현  
- **비밀번호 암호 해시화**를 통한 사용자 정보 보안 강화  

데이터 저장소는 **JSON 기반 데이터베이스 구조**로 설계하여 경량 환경에서도 동작하도록 했으며, 
이를 바탕으로 로그인 • 회원가입 •  회원 정보 조회 및 회원 관리 기능을 구현했습니다.

관광지 및 숙소 데이터베이스는 **Google Sheets** 기반으로 구축하여 팀원들이 동시에 병렬 작업을 수행할 수 있도록 협업 환경을 설계했습니다.<br>
데이터는 주기적으로 **CSV 형식으로 추출·정제**한 뒤, 관광지 및 숙소 선택 로직에서 활용할 수 있도록 데이터 연동 파이프라인을 구현했습니다.<br>
이를 통해 데이터 관리의 유연성과 개발 단계에서의 확장성을 모두 확보했습니다.

### 일정 생성
사용자가 입력한 여행 조건(이동 거리, 여행 유형, 예산, 인원 수 등)을 종합적으로 반영하여  
관광지와 숙소를 자동 선별하는 추천 알고리즘을 설계했습니다.<br>
선정 과정에서는 **위치, 사용자 평가 점수, 숙소 등급(호텔 성급), 관광 유형** 등의 다양한 지표를 기반으로 후보를 필터링하고 우선순위를 산정하도록 구현했습니다.

이후 선별된 데이터를 바탕으로 **LLM 기반 일정 생성 로직**을 적용하여 실제 여행 일정 형태로 구조화했으며,  
**googlemaps 연동**을 통해 동선 정보를 포함한 일정 HTML 파일을 자동 생성하도록 파이프라인을 구축했습니다.<br>
생성된 일정 페이지는 사용자가 즉시 접속·열람할 수 있도록 반환 구조까지 설계하여, 추천 → 일정화 → 시각화로 이어지는 엔드투엔드 흐름을 완성했습니다.
